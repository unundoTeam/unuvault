# Incident Rehearsal Template

> Use one copy per executed rehearsal. This template is not execution evidence.

## Status And Scope

- Rehearsal execution status: `open`
- Branch or exact commit: `<open>`
- Rehearsal date and window: `<open>`
- Signal class: `<open>`
- Scenario: `<open>`

## Ownership

- Decision owner: `<open>`
- Primary responder: `<open>`
- Backup responder: `<open>`
- Acknowledgement target: `<open>`
- Escalation path: `<open>`

## Execution Record

Record timestamps and non-secret evidence for:

1. detection
2. acknowledgement
3. consumer-local, shared-identity, product-data, client-security, or
   launch-packet classification
4. mitigation, rollback, or launch-hold decision
5. recovery confirmation
6. closeout and follow-up assignment

Do not paste raw URLs with query values, headers, tokens, passwords, vault
payloads, ciphertext, callback payloads, service-role keys, or production data.

## Result
